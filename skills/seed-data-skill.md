# [Skill] カレンダー統計データ検証用のテストデータ注入

この Skill は、「フォーカス統計」のカレンダー表示および集計機能が正しく動作するか検証するために、過去7日間のダミーのポモドーロ完了履歴（セッションデータ）を一括生成・シードする手順です。

## 前提条件
- アプリケーションがローカル環境またはVercel本番環境で起動していること。
- クラウド同期のテストを行う場合はログイン済みであること（ローカルのみのテストなら未ログインでも可能）。

## シード実行手順

1. アプリケーション画面の右上にある **「設定」ギアボタン** をクリックして設定モーダルを開きます。
2. モーダルの一番下にある「データ管理」セクションまでスクロールします。
3. **「ダミー統計データを生成 (7日分)」** ボタンをクリックします。
4. これにより、過去7日間のそれぞれの日付に対して、ランダムな回数（1〜3セッション）、ランダムな集中時間（15分, 25分, 50分）の完了セッションログが生成されます。
   - ログイン中の場合は Supabase の `focus_sessions` テーブルに直接インサートされます。
   - 未ログインの場合はブラウザの `localStorage` (キー名: `focusflow_sessions`) に追加されます。
5. 生成完了後、モーダルを閉じて「フォーカス統計」セクションにスクロールします。
6. カレンダーの日付選択ピッカーを操作し、過去の各日付を切り替えて、その日の作業回数および合計作業時間が正しく変化することを確認します。
